## 1. 创建新的 GitHub 仓库

先在 GitHub 上创建一个新的仓库，并复制仓库地址。后面本地项目关联远程仓库时会用到这个地址。

![创建 GitHub 仓库并复制仓库地址](/assets/posts/github-upload-guide/image1.png)

## 2. 打开本地项目目录

在本地找到需要上传的项目文件夹，右击文件夹，选择 `Git Bash Here` 进入命令行操作。

如果电脑还没有安装 Git，需要先到 [Git 官网](https://git-scm.com/install/) 下载并安装。

![在项目目录打开 Git Bash](/assets/posts/github-upload-guide/image2.png)

## 3. 初始化本地 Git 仓库

在命令行中输入下面的命令，在当前项目目录中生成本地 Git 仓库：

```bash
git init
```

![执行 git init](/assets/posts/github-upload-guide/image3.png)

## 4. 添加项目文件

输入下面的命令，将项目中的所有文件添加到本地仓库暂存区：

```bash
git add .
```

![执行 git add](/assets/posts/github-upload-guide/image4.png)

## 5. 提交本地更改

输入下面的命令提交代码。双引号里的 `first commit` 是本次提交的说明，可以根据自己的需要修改：

```bash
git commit -m "first commit"
```

![执行 git commit](/assets/posts/github-upload-guide/image5.png)

## 6. 关联 GitHub 远程仓库

接着使用 `git remote add origin` 将本地仓库关联到 GitHub。

HTTPS 地址示例：

```bash
git remote add origin https://github.com/Counter-Attack-Mankind/Stickman-Parkour-Game.git
```

SSH 地址示例：

```bash
git remote add origin git@github.com:Counter-Attack-Mankind/Stickman-Parkour-Game.git
```

也可以换成自己的仓库地址，例如：

```bash
git remote add origin git@github.com:Counter-Attack-Mankind/embodyfootprint_smooth.git
```

## 7. 推送代码到 GitHub

输入下面的命令，将项目上传到刚创建的 GitHub 仓库中：

```bash
git push -u origin master
```

![执行 git push](/assets/posts/github-upload-guide/image6.png)

如果推送时遇到网络或 HTTP 版本相关问题，可以先尝试切换 Git 的 HTTP 版本：

```bash
git config --global http.version HTTP/1.1
git push origin master
```

如果使用 SSH 推送，命令同样可以写成：

```bash
git push -u origin master
```

如果需要强制覆盖远程仓库，可以使用：

```bash
git push -u origin master --force
```

强制推送会覆盖远程分支历史，使用前要确认远程仓库里的内容可以被替换。
